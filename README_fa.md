# Slipnet Balanced Client
[ [English](https://github.com/docfarzad/Slipnet-Balanced-Client/tree/main) |‌ [فارسی / Persian](https://github.com/docfarzad/Slipnet-Balanced-Client/blob/main/README_fa.md) ]

یک ابزار ساده دسکتاپ برای پیدا کردن DNS Resolver های سالم برای کانکشن های تحت پشتیبانی `slipnet.exe` و سپس اجرای یک پروکسی منسجم به صورت محلی بر پایه اتصال هایی که سالم هستند.

---

## این ابزار چه کار می‌کند

- یک فایل متنی شامل IPهای Resolver را می‌خواند (هر خط یک IP). خطوط تکراری به‌صورت خودکار حذف می‌شوند.  
- آن‌ها را به‌صورت همزمان (parallel) بررسی می‌کند و فقط موارد سالم را نگه می‌دارد.  
- به شما اجازه می‌دهد انتخاب کنید کدام Resolver های سالم فعال شوند.  
- سرورهای پروکسی محلی را اجرا می‌کند:
  - SOCKS5 روی `your_local_ip:1080`
  - HTTP روی `your_local_ip:8080`
- ترافیک را از طریق Resolver های فعال با لود بالانسینگ ساده و مدیریت خطا (cooldown) عبور می‌دهد.

## پیش‌نیازها

- ویندوز (چون نیاز به slipnet.exe دارد، که از [اینجا](https://github.com/anonvector/SlipNet/releases/) قابل دانلود است.)  
- پیشنهاد: Python نسخه 3.9 یا بالاتر  
- پکیج پایتون: requests  
- فایل slipnet.exe باید در همان پوشه فایل Slipnet Balanced Client.py باشد  

نصب نیازمندی:

```bash
pip install requests
```

## نحوه استفاده

1. به راحتی. نسخه آماده استفاده را از [اینجا](https://github.com/docfarzad/Slipnet-Balanced-Client/releases) دانلود و اجرا کنید. یا اگر برنامه نویس هستید، میتوانید در پایتون کد منبع را اجرا کنید:
```bash
python "Slipnet Balanced Client.py"
```
2. در محیط برنامه روی Browse IP List کلیک کنید و فایل لیست Resolver ها را انتخاب کنید.  
3. مقدار Slipnet connection string خود را وارد کنید.  
4. مقدار Workers را تنظیم کنید و روی Start Scan کلیک کنید.  
5. در حین یا پس از پایان اسکن، Resolver های سالم را انتخاب کنید.
6. روی Activate Selected کلیک کنید.
7. استفاده از پروکسی:
   - SOCKS5: your_local_ip:1080
   - HTTP: your_local_ip:8080

8. اختیاری: روی **Verify Good Resolvers' Quality** کلیک کنید تا برای تست فشار (stress test) اجرا شوند و موارد ناپایدار علامت‌گذاری شوند.
9. اگر Resolver های جدیدی را تیک زدید یا از حالت انتخاب در آوردید، دکمه Activate Selected را مجددا بزنید تا تغییرات به مجموعه ریزالور های پشت صحنه اعمال شوند.

## فرمت فایل Resolver

- فایل متنی ساده  
- هر خط یک Resolver  
- خطوط خالی نادیده گرفته می‌شوند  
- خطوط تکراری حذف می‌شوند  

مثال:
```txt
1.1.1.1
8.8.8.8
9.9.9.9
```

---

## توضیحات رفتاری

- نتیجه اسکن شامل latency و وضعیت ترافیک جاری در لحظه است.  
- Resolverهای خراب حذف نمی‌شوند، فقط علامت‌گذاری می‌شوند  
- سیستم به‌صورت خودکار Resolver های ناپایدار را موقت غیرفعال می‌کند. 
- Resolverهای سالم دوباره به‌تدریج برمی‌گردند  
- هنگام بستن برنامه، همه اتصال‌ها به‌درستی بسته می‌شوند.

## ساخت از سورس

1. به پوشه (دایرکتوری) پروژه بروید  
2. برنامه را بسازید (Build کنید)
   
```bash
pyinstaller --onefile --noconsole --name "Slipnet Balanced Client" --add-binary "slipnet.exe;." "Slipnet Balanced Client.py"
```

## رفع مشکل

- executable پیدا نشد → slipnet.exe کنار فایل باشد  
- Resolver سالم نیست → لیست را بررسی کنید  
- پروکسی کار نمی‌کند → پورت‌ها باز باشند  
- اسکن کند است → تعداد Worker را تنظیم کنید
